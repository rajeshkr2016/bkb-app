# Fixes Applied (2026-03-29)

## Supabase Auth 500 Errors
- Identified GoTrue 500s caused by `NULL` values in `auth.users` string columns (e.g., `confirmation_token`, `email_change`).
- Normalized existing rows by setting `NULL` values to empty strings for these columns:
  - `confirmation_token`
  - `recovery_token`
  - `email_change_token_new`
  - `email_change_token_current`
  - `email_change`
  - `phone_change`
  - `phone_change_token`
  - `reauthentication_token`
- Set default values to empty strings for the same columns to prevent new `NULL`s.
- Restarted Supabase containers:
  - `supabase_auth_bkb-app`
  - `supabase_kong_bkb-app`

## Web Chat Navigation
- Added a web-only top header on chat detail with a Back button.
- Back button now always routes to `/chat` on web to avoid history issues.
- Added a web-only bottom navigation bar on chat detail with links to:
  - `Discover`, `Matches`, `Chat`, `Profile`
- Adjusted input row spacing to avoid overlap with the fixed web nav bar.
